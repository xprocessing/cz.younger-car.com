# 2026-01-16 修复模型文件路径错误

## 问题概述

修复了系统中出现的模型文件路径错误，导致系统出现以下错误：

```
Warning: require_once(/www/wwwroot/cz.younger-car.com/includes/database.php): Failed to open stream: No such file or directory in /www/wwwroot/cz.younger-car.com/admin-panel/models/User.php on line 2
```

## 核心问题

模型文件中使用了错误的路径引用，将database.php文件的路径指向了项目根目录下的includes目录，而实际上该文件位于admin-panel/includes目录下。

## 修复内容

### 1. 更新模型文件中的路径引用

**修改了14个模型文件**，包括：
- User.php
- AIGC.php
- InventoryDetails.php
- Costs.php
- OrderProfit.php
- CarData.php
- InventoryDetailsFba.php
- Warehouse.php
- Products.php
- Store.php
- NewProduct.php
- Yunfei.php
- Product.php
- Permission.php
- Role.php

**修改内容**：将所有使用`APP_ROOT`引用includes目录的代码改为使用`ADMIN_PANEL_DIR`，例如：

```php
// 原代码
require_once APP_ROOT . '/includes/database.php';

// 修改后
require_once ADMIN_PANEL_DIR . '/includes/database.php';
```

## 修复结果

✅ **已修复所有模型文件**：确保所有模型文件都能正确引用database.php文件
✅ **解决了路径错误**：不再出现"Failed to open stream"的错误
✅ **保持了代码一致性**：所有模型文件现在使用相同的路径引用方式

## 后续建议

1. 在添加新的模型文件时，确保使用正确的路径引用
2. 定期检查代码中的路径引用，确保它们与项目结构一致
3. 考虑使用自动加载机制（如composer）来避免手动路径引用问题
