# 修复车型数据导入重复条目错误

## 更新内容

1. **问题描述**：在车型数据导入过程中，当遇到重复的 `make-year-trim` 组合时，系统抛出 "Duplicate entry" SQL 错误，导致整个导入过程中断。

2. **解决方案**：修改了 `CarData` 模型中的 `import` 方法，将普通的 `INSERT` 语句改为 `INSERT IGNORE`，这样在遇到重复条目时会忽略该行数据，而不会中断整个导入过程。

3. **修改的文件**：
   - `admin-panel/models/CarData.php`：第 173 行，将 `INSERT INTO` 改为 `INSERT IGNORE INTO`

4. **技术细节**：
   - 使用 `INSERT IGNORE` 语句可以忽略因违反唯一键约束（如 `uk_make_year_trim`）而产生的错误
   - 该解决方案保持了代码的简洁性，对现有逻辑影响最小
   - 不会影响导入的性能，反而会使导入过程更加稳定

5. **测试情况**：
   - 导入包含重复条目的 CSV 文件时，系统不再抛出错误
   - 重复条目被自动忽略，非重复条目正常导入
   - 导入过程能够完整执行完成

## 注意事项

- 使用 `INSERT IGNORE` 会忽略所有因违反约束而产生的错误，不仅仅是重复键错误
- 如果需要更精细的控制，可以考虑使用 `ON DUPLICATE KEY UPDATE` 来更新现有记录
- 建议在导入前对数据进行预处理，尽量避免重复条目
